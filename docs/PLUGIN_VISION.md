# Plugin Vision

The plugin should not merely convert Color Styles into Variables.

Its purpose is to reconstruct a modern token architecture from an existing Design System.

Pipeline:

Color Styles
+
Text Styles
+
Component Context
+
Layer Names
+
Usage Analysis

=> Semantic Inference
=> Review Queue
=> Variables
=> Migration

Target outcome:

Primitive Tokens
Semantic Tokens
Usage Tokens

with components consuming Usage Tokens.
